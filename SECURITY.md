# Security Policy

## Supported versions
Only the latest release line is actively supported.

## Reporting a vulnerability
Do not open public issues for security vulnerabilities.

Please use GitHub private vulnerability reporting (Security Advisories) for this repository.
Include:
- Affected version or commit
- Reproduction steps
- Impact assessment
- Any proposed fix

## Response targets
- Initial acknowledgement: within 72 hours
- Triage decision: within 7 days
- Fix timeline: depends on severity and scope

## Disclosure
We follow coordinated disclosure. Please allow maintainers time to patch before public disclosure.
